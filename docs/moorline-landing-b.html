Origin — how MOORLINE came to be
canon is the soul

The session that broke canon

Canon is the soul. do→define→refine is not ceremony — it is the machine's heartbeat. Canon carries the persistent identity of a project: consulted at arc open, refined at arc close. The loop is the truth.

A long session drifted. Execution machinery was exciting — worktrees, the Orca app, GitHub PRs, hook automations. By the time the session ended, control-home CANON.md had empty top sections and duplicate journal entries. Canon was broken and un-updated. Nobody noticed until it was too late to reconstruct the reasoning.

Mechanism beats willpower. Discipline fails under deadline pressure, excitement, and fatigue. The old self-consistency check asked the wrong question: "does CANON.md's sha agree with its own stamps?" could not see that reality had outrun canon. What does not exist as a gate does not exist as a constraint.

Synthesis. A 4-lens + adversarial-critique workflow produced six mechanisms. The keystone: reality-rev — a git-tree sha over the paths canon claims to cover — finally makes "code outran canon" a detectable, gate-blocking state. That is what every prior model missed.

§1 — Core Principle

The single door every unit of work passes through

Canon is structurally central when the definition of work is the only door work can pass through — at birth, at every transition, and at the exit — and that door is a single, two-axis, fail-loud oracle that no role, no interactivity, and no automation can route around. Every work transition calls this one oracle. Mechanism, not willpower: there is no green path that leaves canon behind.

fig.1 — tide cannon gate: the single two-axis oracle every work transition calls

The gate is a real subcommand backed by src/tide/gate.py, POSIX exit code so it composes verbatim in shells, hooks, CI, and Orca --precheck. Every other gate delegates here — one definition of "current," nothing can drift from it. Tri-state is load-bearing: callers must treat 2 as FAIL-LOUD, never as skip, so a broken oracle cannot silently disable enforcement.

§2 — The Two Axes

cannon-rev × reality-rev — the keystone blind-spot fix

The fatal blind spot: asking "does CANON.md's sha agree with its own stamps?" is the wrong axis. The neglect we are preventing is reality outrunning canon. Two revs, carried everywhere, make that detectable.

§3 — The Six Mechanisms

One machine — six load-bearing components

Each mechanism is a structural piece. Remove one and canon-neglect re-becomes possible in that dimension. Together they make neglect technically unrepresentable, not merely discouraged.

fig.3 — machine overview: six components, one oracle at center
M1 tide cannon gate — the single two-axis oracle
gate.decide(root) → (code, reasons) clauses: (a) no unmerged deltas (b) no arc drifted on cannon-rev OR reality-rev (c) cannon lint passes (structural health) (d) no arc dirty without substantive delta (e) no open arc with empty canon basis (f) no unpaid canon-debt (open force/noop records) 0 = current 1 = stale 2 = oracle-error (FAIL-LOUD, never skip)
Enforced src/tide/gate.py — POSIX exit code, composes in shells / hooks / CI / Orca --precheck. Every other gate delegates here. Canary cron stales a scratch project and asserts the gate trips. Prevents The root neglect — nothing forced attention back to canon. No progress path is green while canon is stale. A silently-dead oracle is itself detected (state 2).
M2 Two-axis freshness — reality-rev
CANON.md carries: canon-covers: [path/globs...] reality-rev = git-tree-sha(those globs) tide arc new → stamps cannon-rev + reality-rev into arc frontmatter tide arc close → re-stamps both gate fires STALE when: reality-rev moved and merged delta touched no canonical section (## What it is / ## State & components / ## Interfaces)

M4 Atomic serialized merge — safe under Orca parallelism

§4 — Orca-Native Execution

The binding — two revs keyed across every surface

One arc ⇄ one worktree ⇄ one GitHub issue ⇄ one PR ⇄ one contract ⇄ one cannon-delta, all keyed by both cannon-rev and reality-rev stamped at birth. tide stays the METHOD. Orca stays the MECHANISM. The two revs are the single binding that makes neglect technically impossible.

Bind at birth
tide cannon gate → called BEFORE gh / worktree side-effects → owed delta = no issue, no worktree
arc ⇄ worktree ⇄ issue ⇄ PR ⇄ contract ⇄ delta
tide arc new --issue → arc frontmatter + .tide/state/arc-rev + issue body → cannon-rev + reality-rev stamped
Orca native "new worktree" button cannot be removed, so: tide cannon gate run as required PR status check → independently re-derives arc⇄issue⇄rev binding from PR branch → fails any PR whose worktree was not born through tide arc new (un-stamped work can commit, but never reaches terminal state) PostToolUse hook refuses writes inside a worktree lacking valid arc-rev. Orca automations created with --precheck 'tide cannon gate --project <p>' → cron firing on stale canon is skipped → on canon-debt: opens tracked issue, pushes notification, after N aborted cycles auto-spawns reconcile-canon arc Terminal state: tide arc close → trunk-only cannon merge → bumps both revs Branch protection: include-administrators ON + server-side pre-receive ruleset → solo owner's force-merge still runs the gate → pre-receive rejects pushes that "Closes #N" while arc owes a delta

§5 — Stop Points (this session, in order)

How MOORLINE catches the exact failure that produced it

Concrete moment: humans + agent got excited about Orca / worktree / issue / PR machinery, left cannon-merge broken (empty top sections, duplicate journal entries), and forgot to update canon after the changes.

  1. M6 — home scope gate
    The Orca / hooks / PR work touched tide's own code and control-home. M6 makes that arc-shaped — the pre-push gate would have refused to merge the execution-machinery changes until the home canon's delta was reconciled. The work could not have shipped past canon.
  2. M2 × M5 — reality-rev trips STALE, pulse surfaces it
    Reality moved (lots of new code) while CANON.md's top sections stayed empty. M2's reality-rev trips STALE ("canon has not absorbed the last N commits"), which M1 surfaces as non-zero on the next arc-open and on the SessionStart banner (M5 canon-pulse, first text on screen).
  3. M4 × M5 — corruption is non-representable at write time
    Empty top sections / duplicate journal entries could not have been a resting state. M4's idempotent + locked + self-linting merge refuses to produce duplicate journal entries or swap in a file that fails lint. M5's PostToolUse + pre-commit would have rejected the corrupting write at the instant it happened.
  4. M3 — even force cannot produce a clean exit
    Even under deadline pressure, closing with a stub or a -f would not clear the gate. M3 demands a canonical-section delta or a signed noop that itself becomes gate-blocking canon-debt. There was no green path that left canon behind.
§6 — Build Order

Fastest path to canon-central

Dependency-ordered. Each step unblocks the next.

  1. cannon lint + idempotent / locked / atomic merge (M4 + M5b)
    Cause-fix the confirmed duplicate-journal bug. Make corruption non-representable. Smallest, highest-leverage step; unblocks everything downstream. Add per-entry content-hash dedup, flock + os.replace + CAS.

  2. tide cannon gate tri-state oracle (M1)
    One gate.py, POSIX exit. This is the keystone every later call site reuses. Wire clauses (a)→(f); add the canary-cron test.
    src/tide/gate.py
  3. reality-rev two-axis freshness (M2)
    Add canon-covers: manifest + reality-rev stamping; wire into the gate. This is what actually fixes the original neglect direction.
    src/tide/cannon/rev.py · src/tide/cannon/reality.py · src/tide/sync.py
  4. DEFINE state + substance / consultation guard + force-debt (M3)
    Make the loop's front and back halves structural. Insert DEFINE into the contract state machine; add canon basis pre-fill at arc-new; make force write gate-blocking canon-debt (not keyed on interactivity).
    src/tide/contract/lifecycle.py · src/tide/arc/stream.py
  5. Hooks: canon-pulse banner + PostToolUse guardian + pre-commit ban (M5a / M5c / M5d)
    Cheap, high-visibility; closes the write-time hole. Reorder session_start so canon block is the first text on screen.
    hooks/session_start.py · hooks/edit_gate.py
  6. Orca wiring + M6 home-as-project gate
    Required PR check re-derives the binding; automation precheck with escalation; terminal-state binding; include-administrators + pre-receive hook; home canon gate. Last because it depends on the oracle (step 2) existing and being trustworthy.
